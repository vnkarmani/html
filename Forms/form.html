Html Form

An HTML form is used to collect user input.

The user input is sent to a server for processing.

The HTML <form> element is used to create an HTML form for user input

<form>----</form>

The <form> element is a container for different types of input elements, such as: text fields, checkboxes, radio buttons, submit buttons, etc.

Form attributes:

1.Action

The "action" attribute defines the action to be performed when the form is submitted.

Usually, the form data is sent to a file on the server when the user clicks on the submit button.

the form data is sent to a file which we write through "action" attribute.

2.Method

The method attribute specifies the HTTP method to be used when submitting the form data.

The default HTTP method when submitting form data is GET.

the form data sent to the URL when we use GET method,so get method is less secure.

using GET method,the submitted data is visible on url

in GET method,The length of a URL is limited.

GET is good for non-secure data, like query strings in Google.

using POST method,,the submitted data is not visible on url.

in POST method,The length of a URL is not limited(unlimited),so it is used to send large amounts of data

POST method is more secure then GET.

So,Always use POST if the form data contains sensitive or personal information.

3.Autocomplete

The autocomplete attribute specifies whether a form should have autocomplete on or off.

When autocomplete is on, the browser automatically complete values based on values that the user has entered before.

by dedault,its value is off.

4.nonvalidate

When nonvalidate is present, it specifies that the form-data (input) should not be validated when submitted.

5.enctype

it Specifies how the form-data should be encoded when submitting it to the server.

it is used only for POST method.
